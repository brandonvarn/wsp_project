Components

Component: Header

The header is the most important aspect to the page. It sets the tone and allows the user to grasp right away what the company is about.

I would build this by using these properties:

  • CSS:
    1. Height
    2. Width
    3. Float
  • Having login information listed in the right corner
  • Having contact information listed on the right below login

Component: Nav

The NAV is probably going to be the most used part of any website. It links to all the main pages and is usually the first place a user will go when they need to browse your website.

If I was to build a NAV right now I would make a unordered list. Then design it through CSS. Make it display the direction I want (vertical or horizontal). Apply any highlight elements or animation. You can even start to add things like jQueary

Component: Table

Tables are used for many things now. Before they were used for styling as well as displaying of information. But with today's CSS3 tables can now be properly used. On almost any business site you will find a table. Usually displaying the benefits of different packages offered. Or displaying what product is the best.

I would build this by creating the table in html with html5 markup. I would then go through with css and style it how I wanted it. For example if I had a list of products I wanted to place down I would place each column with a different color shade. And have the inside's have a border to it.

Component: CTA

A call to action is dependont on color, scale, language, and imagery. You need to have the proper planning and forethought to achieve a well designed CTA. A CTA is mainly used to sell whatever it is you are trying to sell on any page. It can be rock climbing by showing a mountain top with a man on top. Or it can be a great sale on beauty products being displayed.

If I was to build a CTA I would first sit down and decide how I would want it to look. I would then decide if it is purely text based (most are not). Once that has been decided I would use CSS to make sure it has the look I want.

Component: Footer

I have started to notice a lot in responsive sites that the footer is more and more important. The footer can contain very important information like a home button or disclaimers. The footer has many uses it just depends on what kind of site you are building. For our personal company site we use the footer for all our main links (a lot of this has to do with making sure keywords are used enough times and appropriately) as well as a disclaimer.

I would build the footer a lot like the nav. I would line the links on the bottom and probably but the years the business has been doing business.

Featured Component

Choice

I chose the header for my featured item. I am using the header from First National Bank or FNB. FNB is a banking site that uses the header image above it's main content. You can see the link bar as well as the login information shown. The border is 0 around the image and the width would be 100% in this case it was set at 960 px.

Font's and Colors

The font used is Sans Serif. No google font's or any other loaded font is used. So they took a very minimalist approach to the design. The font color used is white on everything.

CSS

The header is actually a image with div tag's placed above it for the login and link bar. The image has no border and is 960px wide and 260px tall.
